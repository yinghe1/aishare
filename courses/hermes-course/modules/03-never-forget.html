Never Forget

How Hermes builds a profile of you — and gets smarter every session

Three Kinds of Memory

🧠
Most AI chatbots forget you the moment you close the tab.

Hermes builds a profile of who you are, what you know, how you work — and gets better every single session.

Before every conversation starts, Hermes reads three memory stores and loads them into the system prompt. By the time you type your first message, Hermes already knows you.

🗒️
MEMORY.md

Hermes’ personal notebook. Environment facts, project conventions, tool quirks, things it noticed about your setup. Hermes writes to this file; you can read and edit it any time.

👤
USER.md

Your profile. Preferred coding language, communication style, expertise level, workflow habits. Hermes updates this as it learns more about you — or you can edit it directly to tell Hermes exactly who you are.

📋
Skills Library

Reusable step-by-step procedures for complex tasks. Each Skill is a plain text file Hermes (or you) can write and edit. Loaded on demand so they don’t bloat memory.

💡
All three load before the conversation begins.

Hermes doesn’t need you to re-introduce yourself. It starts already knowing your preferences, your environment, and which procedures it has mastered.

The Frozen Snapshot

Memory is read once at session start and “frozen” into the system prompt. Writes during a session update the files on disk immediately — but the AI doesn’t see them until next session. Here’s why that’s actually clever.

Why freeze it? Speed.

Keeping the system prompt stable across the whole session lets the AI model use prefix caching — it processes the opening context once and reuses the result for every follow-up message. Stability beats real-time updates.

🔒
What about mid-session writes?

When Hermes learns something important during your conversation, it writes to disk immediately — durable and safe. But the current session continues from the frozen snapshot. The new knowledge takes effect next session.

Skills — Reusable Superpowers

Skills are plain text files with a specific format. They use frontmatter at the top (written in YAML) followed by full instructions. The clever part: Hermes uses progressive disclosure to keep sessions lean.

CODE 

"""
Skills Tool Module

Inspired by Anthropic's Claude Skills system
with progressive disclosure architecture:
- Metadata (name ≤64 chars, desc ≤1024)
  shown in skills_list
- Full Instructions — loaded via
  skill_view when needed
- Linked Files (references, templates)
  loaded on demand

Directory Structure:
    skills/
    ├── my-skill/
    │   ├── SKILL.md     # required
    │   ├── references/
    │   └── assets/
"""
PLAIN ENGLISH

Skills are step-by-step procedure files — like recipe cards Hermes can follow.

The design is inspired by how Anthropic built skills for Claude, but extended for Hermes.

Tier 1 — Name + description only: tiny. This is what Hermes sees in its skills list at session start. Stays within a strict character budget to save tokens.

Tier 2 — Full instructions: loaded only when you invoke the skill with a slash command.

Tier 3 — Linked files: reference docs and templates, loaded on demand if the skill needs them.

Each skill lives in its own folder. The SKILL.md file is the required entry point — everything else is optional.

Four Types of Skills

🏠

Builtin Skills

Pre-installed with Hermes — always available the moment you start. Core procedures like code review, research templates, and writing guides.

📦

Optional Skills

Included in the Hermes repo but not active by default. Enable them when you need them — specialized workflows that not everyone uses.

🌐

Hub Skills

Community-built skills from GitHub or agentskills.io. Install with one command. Someone else already solved that hard problem — use their skill.

✏️

Custom Skills

Write your own SKILL.md files. Plain markdown — no coding required. If you have a workflow you repeat, turn it into a skill and Hermes handles it perfectly every time.

Skill Metadata Fields

Every SKILL.md starts with a frontmatter block. These fields control how Hermes finds, identifies, and restricts each skill.

name
Max 64 characters

What you type after the slash — like /axolotl-finetuning. Short, memorable, no spaces.

desc
Max 1,024 characters

Shown in the skills list — this is the only part Hermes reads at session load. Keep it crisp.

ver
Optional version tag

Like 1.0.0 — helps you track when you last updated a skill and compare with hub versions.

plt
Optional platform filter

Values: macos, linux, windows. Omit to load everywhere. A macOS-only skill won’t appear on your Linux server.

The Learning Loop

Hermes doesn’t just follow instructions — it writes new ones. After a complex task, it can save what it learned as a Skill so the next session starts smarter. This is how Hermes compounds knowledge over time.

1
You complete a complex task together

You ask Hermes to help set up a GPU training cluster. It figures it out, step by step, with you.

2
Hermes offers to save the procedure

At the end, it says: “Should I save this as a skill so we skip the trial-and-error next time?” You say yes.

3
Skill saved to ~/.hermes/skills/

The procedure is written as a SKILL.md file in your personal skills folder. Plain text — fully readable and editable by you.

4
Next session: skill loads automatically

At startup, Hermes scans the skills folder and adds the name and description to the system prompt. It knows the skill exists.

5
Hermes handles this task efficiently from now on

Next time you mention GPU training, Hermes invokes the skill and follows the exact procedure that worked before — no guessing required.

Glossary

system prompt

The background instructions the AI reads before every conversation — like a briefing document.

prefix cache

Reusing the AI model’s work from the stable beginning of each prompt — major speed gain.

progressive disclosure

Show name + description now. Load full instructions only when invoked. Keeps memory lean.

token

The unit of text an AI model processes — roughly 3/4 of a word. Token budgets shape every design decision.

frontmatter

Structured settings at the top of SKILL.md — name, version, description, platform restrictions.

YAML

The format used in skill frontmatter. Reads like plain English, parsed precisely by machines.

FTS5

The search engine built into SQLite that Hermes uses to search past conversation sessions by keyword.
