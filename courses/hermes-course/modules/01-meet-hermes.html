What Is Hermes?

⚡
You typed a message and hit send.

In the next few seconds, Hermes picked up your meaning, spun up a terminal, browsed the web, wrote some code, and replied — all without you writing a single line. Here's exactly what happened.

Hermes is an AI agent: an artificial intelligence that can act in the world, not just talk about it. Think of it as a brilliant personal assistant who never forgets, never sleeps, and lives wherever you need them — your laptop, your cloud server, your Telegram app.

🤖
AI Agent

Not just a chatbot — Hermes takes real actions: runs commands, reads files, browses the web, writes code, and delegates work to copies of itself called subagents.

💻
Terminal Interface

You chat with Hermes inside a real terminal using a TUI — a keyboard-driven chat interface that feels like messaging, but runs on raw compute power.

🧠
Self-Improving

After complex tasks, Hermes writes what it learned as a reusable Skill. Each session starts smarter than the last.

🌍
Runs Anywhere

Your laptop, a VPS, a Docker container, or a serverless cloud function — Hermes has six execution backends, so it goes where the work is.

📱
Multi-Platform

Not just the terminal — Hermes can receive messages from Telegram, Discord, Slack, and more via its built-in Gateway.

Your First Session, Traced

You open a terminal and type hermes. You ask: "Set up a Python project, install Flask, write a hello world app." Here's what happens — step by step — in the next few seconds.

1
TUI shows your message

The TUI captures your message and displays it in the conversation panel.

2
AIAgent reads your request

The AIAgent receives your message along with its full context — past conversation, available tools, and any active skills.

3
AIAgent checks its Skills library

Before doing anything new, Hermes checks its Skills library — maybe it has done this exact setup before and can reuse a known-good procedure.

4
AIAgent calls terminal tools to create files

Hermes runs real shell commands: mkdir, pip install flask, writes app.py — all through its toolset.

5
AIAgent browses docs if needed

If Flask's API has changed or Hermes is unsure of a detail, it fires a web search and reads the official documentation in real time — not from outdated training data.

6
AIAgent returns the result

Once everything is done, Hermes sends a summary back to your TUI — what it created, what commands ran, and what to do next.

🔁
The Core Loop

Ask → Think → Act → Repeat. This loop runs dozens of times per response. Every tool call, every web search, every file write is one trip around the loop.

The Product Tour

Every Hermes capability is a subcommand in your CLI. Here's the full menu:

hermes Open the interactive TUI chat — your main workspace with Hermes
hermes gateway Start the Gateway — receive messages from Telegram, Discord, Slack, and more
hermes cron Manage scheduled tasks — run Hermes on a timer, even while you sleep
hermes model Switch which AI model Hermes uses for reasoning — no other changes needed
hermes tools Configure which tools are enabled — allow or restrict what Hermes can do
hermes skills Browse and manage the Skills library — see what Hermes has learned across past sessions

The CLI Manifest — Code ↔ Plain English

This is the __init__.py file — the front page of the Hermes CLI package. Here's what each line means:

CODE 

"""
Hermes CLI - Unified command-line interface for Hermes Agent.

Provides subcommands for:
- hermes chat          - Interactive chat
- hermes gateway       - Run gateway in foreground
- hermes gateway start - Start gateway service
- hermes gateway stop  - Stop gateway service
- hermes setup         - Interactive setup wizard
- hermes status        - Show status of all components
- hermes cron          - Manage cron jobs
"""

__version__ = "0.10.0"
__release_date__ = "2026.4.16"
PLAIN ENGLISH

The triple-quoted block at the top is a docstring — a human-readable description of what this file (or program) does. Python ignores it at runtime; it exists purely to help developers (and Hermes!) understand the code.

hermes chat — opens the interactive TUI; this is the same as just running hermes with no extra words.

hermes gateway — starts the bridge that lets Hermes receive messages from Telegram, Discord, or Slack. Running it "in foreground" means you can see its logs directly in your terminal.

hermes gateway start/stop — runs the Gateway as a background service, like a web server that keeps going even after you close your terminal.

hermes setup — a guided wizard that asks you questions and configures Hermes for your environment. Perfect for first-time setup.

hermes status — a health check: shows you whether each part of Hermes (Gateway, scheduler, model connection) is running correctly.

hermes cron — manage scheduled tasks: tell Hermes to run a specific prompt every morning, every hour, or on any schedule you define.

__version__ and __release_date__ — machine-readable labels for the current version. Other parts of the code read these to show you version info or check for updates.
