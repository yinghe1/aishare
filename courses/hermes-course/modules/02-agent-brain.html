The Loop, Demystified

When you ask Hermes to find the five fastest Python frameworks, it doesn’t just answer from memory. It browses, reads, compares — and the whole loop runs dozens of times in seconds. Here is the engine behind that.

Think of it like a chess grandmaster’s thinking process: look at the board, pick the best move, make it, look again — repeat until the game is won. Each round is called an iteration.

Parallel Tool Calls

Hermes can call multiple tools at the same time — like searching the web and reading a local file simultaneously. But not every tool is safe to parallelize.

🔍
Web Search — parallel-safe

Pure read operation: fetches data from the internet, changes nothing. Multiple searches can run at the same time without interfering with each other.

📄
File Read — parallel-safe

Reading a file has no side effects. Ten file reads can run at once; none will affect the others.

💻
Terminal — sequential only

Shell commands can conflict — two commands modifying the same file simultaneously could corrupt it. Terminal commands always run one at a time to prevent collisions.

🙋
Clarify — runs alone

When Hermes needs to ask you a question, it must wait for your answer before doing anything else. This tool always runs by itself, never in parallel.

The Safe List — Code ↔ Plain English

Hermes maintains a hardcoded list of tools that are safe to run simultaneously. Here is that list:

CODE 

_PARALLEL_SAFE_TOOLS = frozenset({
    "ha_get_state",
    "ha_list_entities",
    "ha_list_services",
    "read_file",
    "search_files",
    "session_search",
    "skill_view",
    "skills_list",
    "vision_analyze",
    "web_extract",
    "web_search",
})
PLAIN ENGLISH

_PARALLEL_SAFE_TOOLS is a frozenset — an immutable (unchangeable) collection of tool names. Frozen means the code cannot accidentally modify this list at runtime. It is a safety guarantee baked into the language.

ha_get_state, ha_list_entities, ha_list_services — Home Assistant integration tools. They read the current state of your smart home devices (lights, thermostats, etc.) without changing anything. Read-only = parallel-safe.

read_file, search_files — read a specific file, or search for files matching a pattern. No writes, no deletions — safe to do many at once.

session_search — searches the current conversation history for previous messages or context. Pure lookup, no changes.

skill_view, skills_list — look at Hermes’ saved Skills library. Reading a recipe does not change it.

vision_analyze — sends an image to the AI model for analysis. The image is not modified; only a description comes back.

web_extract, web_search — fetch web pages or run searches. The internet is not changed by you reading it. Both are safe to run simultaneously.

⚡
Why Parallelism Matters

Parallelism is why Hermes can often complete complex research tasks in a fraction of the time a human would take. Six web searches at once instead of six in a row — that is a 6x speedup on the bottleneck.

The Iteration Budget

Each agent gets an energy allowance — a maximum number of iterations before it must stop. This prevents runaway agents from looping forever. Parent agents get 90 by default; subagents get 50.

CODE 

class IterationBudget:
    def __init__(self, max_total: int):
        self.max_total = max_total
        self._used = 0
        self._lock = threading.Lock()

    def consume(self) -> bool:
        """Try to consume one iteration. Returns True if allowed."""
        with self._lock:
            if self._used >= self.max_total:
                return False
            self._used += 1
            return True
PLAIN ENGLISH

class IterationBudget — a blueprint for a “fuel gauge” object. Every agent gets one of these when it starts; it tracks how many iterations have been used and stops the agent when the tank is empty.

max_total — the maximum number of loop iterations this agent is allowed. Set to 90 for parent agents, 50 for subagents.

_used = 0 — a counter that starts at zero and goes up by one every time the agent takes a turn. The underscore prefix signals this is internal — other code should not touch it directly.

_lock = threading.Lock() — a lock that makes the counter thread-safe. Because Hermes runs multiple tools in parallel, two threads might try to update the counter at the same instant. The lock prevents a collision.

consume() — called once per loop iteration. It checks: have we hit the limit? If yes, return False (stop). If no, increment the counter and return True (keep going). The with self._lock block ensures only one thread can do this check-and-increment at a time — no double-spending iterations.

When the Budget Runs Out — What To Do

1
Tell Hermes to continue

Just say “continue from where you left off.” Hermes starts a new session with fresh budget but carries forward the context window summary of what was done.

2
Break the task into smaller pieces

Instead of “build me a full app,” try “set up the project structure,” then “write the database models,” then “add the API routes.” Each piece fits comfortably inside one budget.

3
Increase the budget in config

For long-running automation tasks, you can raise the maximum iteration count in Hermes’ configuration file. This is a power-user option — use it only when you understand why the task needs so many loops.

When Things Go Wrong

Knowing the loop architecture helps you debug Hermes when it gets stuck. Here are the most common failure patterns and how to recognize them:

🔁
The Loop Cycle

Hermes keeps trying the same approach and getting the same error. The fix: interrupt it and give more specific guidance — you are the circuit breaker. The agent sees the same inputs, so it makes the same decisions. Change the inputs.

📏
Context Window Full

After very long sessions, the context window fills up and Hermes starts to “forget” earlier parts of the conversation. Start a new session and give it a concise summary of where you are.

🔌
Provider Issues

If the provider is down or rate-limiting you, Hermes will report tool call failures. Use hermes model to switch to a different model provider instantly.

🔀
API Format Mismatch

Different AI providers send and receive tool call data in slightly different formats. Hermes handles this via its api_mode setting — a translation layer called runtime_provider converts between known formats automatically.

🛠️
You Are Part of the Loop

The agent loop is not just an internal machine — you are a participant. When Hermes gets stuck, your intervention (new context, a different approach, breaking a task smaller) is itself a form of input to the loop that changes its trajectory.
