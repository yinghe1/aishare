Writing Richer Notes

The obsidian-markdown skill — teaching AI to write notes that think, link, and display beautifully.

Standard markdown vs. Obsidian markdown

Regular markdown is like plain text with some formatting. Obsidian markdown is a living document system — notes link to each other, callouts draw the eye, properties make notes searchable, and embeds pull in content from anywhere in your vault.

Without the skill, AI writes the left column. With it, you get the right.

WITHOUT SKILL

# Meeting Notes

See Project Alpha for context.

Warning: deadline is tomorrow.

Tasks:
- Send report
- Call team

WITH SKILL

---
date: 2024-01-15
tags: [meeting, project]
---

See [[Project Alpha]] for context.

> [!warning] Deadline Tomorrow
> Send report by 5pm.

- [ ] Send report
- [ ] Call team
💡
Why this matters for AI steering

When you know these patterns exist, you can tell AI exactly what to create: "add a warning callout," "link this to [[Project Alpha]]," "add a status property." Without the vocabulary, you get vague results. With it, you get precision.

Wikilinks — the postal address system

Think of every note in your vault as a house. A wikilink is the address on an envelope — it tells Obsidian exactly which house to deliver you to. And unlike street addresses, if a house moves, all the envelopes update automatically.

CODE 
[[Note Name]]
[[Note Name|Display Text]]
[[Note Name#Heading]]
[[Note Name#^block-id]]
[[#Heading in same note]]

![[Note Name]]
![[image.png|300]]
PLAIN ENGLISH

Link to a note — clicking jumps there

Same link but shows "Display Text" instead of the note name

Link directly to a specific heading inside the note

Link to a specific paragraph (block) — you mark blocks with ^id

Link to a heading within the same note you are writing

The ! prefix means "embed" — pull the content inline instead of just linking

Embed an image, set its display width to 300 pixels

🔗
Rule of thumb: wikilinks for notes, URLs for the web

Use [[wikilinks]] for anything inside your vault. Use [text](https://...) for external websites. Wikilinks let Obsidian track renames — regular links break when you rename notes.

Callouts — sticky notes that know their mood

A callout is like a sticky note that color-codes itself based on what it contains. A warning turns orange. A tip turns cyan. A success turns green. You just pick the type — Obsidian handles the design.

> [!note]
> Basic note callout.

> [!warning] Custom Title
> Callout with a custom title.

> [!tip]+ Expanded by default
> Foldable callout — + means open, - means collapsed.
📝

note · info · todo

Blue tones. General information, announcements, to-do items.

💡

tip · hint · important

Cyan tones. Helpful suggestions and things worth highlighting.

⚠️

warning · caution

Orange tones. Alerts, gotchas, things to watch out for.

✅

success · check · done

Green tones. Completed items, confirmations, wins.

🐛

danger · bug · failure

Red tones. Errors, bugs, critical issues, missing items.

📋

example · quote · abstract

Purple/gray tones. Examples, citations, summaries.

Properties — the book jacket of your note

A book jacket tells you the title, author, genre, and publication year before you open the cover. Frontmatter does the same for your notes — structured data at the top that Obsidian can search, filter, and display as columns in a database view.

CODE 
---
title: Project Alpha
date: 2024-01-15
tags:
  - project
  - active
status: in-progress
cssclasses:
  - custom-class
---
PLAIN ENGLISH

Start of the properties block — must be first thing in the file

A custom text field — shows up as a column in table views

A date field — Obsidian can sort and filter by this

Tags list — each item becomes a searchable label

Tags can be nested: #project/active, #project/done

Any custom field you invent — "status", "priority", "author" all work

CSS classes to apply custom styling from a snippet file

End of the properties block
