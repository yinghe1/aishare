01

What Are AI Skills?

How a tiny text file turns a generic AI into an Obsidian expert.

Your AI doesn't speak Obsidian — yet

Imagine you ask your AI assistant to "create a note about today's meeting." It writes plain text. No links to related notes. No callouts. No tags. No frontmatter. Just... words in a file.

That's not because the AI is bad. It's because Obsidian has its own language — special syntax that makes notes connect, display beautifully, and become searchable. Without being taught that language, AI produces generic output.

😐

Without Skills

AI writes plain markdown. No wikilinks, no callouts, no properties. Works everywhere, optimized for nowhere.

✨

With Skills

AI writes native Obsidian syntax. Wikilinks, callouts, frontmatter, bases — all correct, all connected.

A skill is a briefing document for your AI

Think of it like a new employee orientation packet. Before someone starts a job, you hand them a guide: "Here's how we write emails. Here's the format we use for reports. Here's what to avoid." A skill file does exactly that, but for AI.

CODE 
---
name: obsidian-markdown
description: Create and edit Obsidian
  Flavored Markdown with wikilinks,
  embeds, callouts, properties...
---

# Obsidian Flavored Markdown Skill
Create and edit valid Obsidian
Flavored Markdown...
PLAIN ENGLISH

The dashes mark the start of the file's identity card (called frontmatter).

"name" is the skill's ID — how other tools refer to it.

"description" tells the AI exactly when to use this skill — it reads this to decide whether it's relevant.

The dashes close the identity card.

Everything after is the actual instructions — rules, examples, syntax the AI should follow.

💡
The "description" field is the trigger

When you type a request, the AI reads skill descriptions to decide which ones apply. If your request matches — "create a wikilink," "make a callout," "add properties" — the AI loads the full skill and follows its rules. The description is like a job listing: it decides who gets called in.

02

Writing Richer Notes

The obsidian-markdown skill — teaching AI to write notes that think, link, and display beautifully.

Standard markdown vs. Obsidian markdown

Regular markdown is like plain text with some formatting. Obsidian markdown is a living document system — notes link to each other, callouts draw the eye, properties make notes searchable, and embeds pull in content from anywhere in your vault.

Without the skill, AI writes the left column. With it, you get the right.

WITHOUT SKILL

# Meeting Notes

See Project Alpha for context.

Warning: deadline is tomorrow.

Tasks:
- Send report
- Call team

WITH SKILL

---
date: 2024-01-15
tags: [meeting, project]
---

See [[Project Alpha]] for context.

> [!warning] Deadline Tomorrow
> Send report by 5pm.

- [ ] Send report
- [ ] Call team
💡
Why this matters for AI steering

When you know these patterns exist, you can tell AI exactly what to create: "add a warning callout," "link this to [[Project Alpha]]," "add a status property." Without the vocabulary, you get vague results. With it, you get precision.

Wikilinks — the postal address system

Think of every note in your vault as a house. A wikilink is the address on an envelope — it tells Obsidian exactly which house to deliver you to. And unlike street addresses, if a house moves, all the envelopes update automatically.

CODE 
[[Note Name]]
[[Note Name|Display Text]]
[[Note Name#Heading]]
[[Note Name#^block-id]]
[[#Heading in same note]]

![[Note Name]]
![[image.png|300]]
PLAIN ENGLISH

Link to a note — clicking jumps there

Same link but shows "Display Text" instead of the note name

Link directly to a specific heading inside the note

Link to a specific paragraph (block) — you mark blocks with ^id

Link to a heading within the same note you are writing

The ! prefix means "embed" — pull the content inline instead of just linking

Embed an image, set its display width to 300 pixels

🔗
Rule of thumb: wikilinks for notes, URLs for the web

Use [[wikilinks]] for anything inside your vault. Use [text](https://...) for external websites. Wikilinks let Obsidian track renames — regular links break when you rename notes.

Callouts — sticky notes that know their mood

A callout is like a sticky note that color-codes itself based on what it contains. A warning turns orange. A tip turns cyan. A success turns green. You just pick the type — Obsidian handles the design.

> [!note]
> Basic note callout.

> [!warning] Custom Title
> Callout with a custom title.

> [!tip]+ Expanded by default
> Foldable callout — + means open, - means collapsed.
📝

note · info · todo

Blue tones. General information, announcements, to-do items.

💡

tip · hint · important

Cyan tones. Helpful suggestions and things worth highlighting.

⚠️

warning · caution

Orange tones. Alerts, gotchas, things to watch out for.

✅

success · check · done

Green tones. Completed items, confirmations, wins.

🐛

danger · bug · failure

Red tones. Errors, bugs, critical issues, missing items.

📋

example · quote · abstract

Purple/gray tones. Examples, citations, summaries.

Properties — the book jacket of your note

A book jacket tells you the title, author, genre, and publication year before you open the cover. Frontmatter does the same for your notes — structured data at the top that Obsidian can search, filter, and display as columns in a database view.

CODE 
---
title: Project Alpha
date: 2024-01-15
tags:
  - project
  - active
status: in-progress
cssclasses:
  - custom-class
---
PLAIN ENGLISH

Start of the properties block — must be first thing in the file

A custom text field — shows up as a column in table views

A date field — Obsidian can sort and filter by this

Tags list — each item becomes a searchable label

Tags can be nested: #project/active, #project/done

Any custom field you invent — "status", "priority", "author" all work

CSS classes to apply custom styling from a snippet file

End of the properties block

03

Notes as a Database

The obsidian-bases skill — teaching AI to turn your vault into queryable, live-updating views.

What if your notes could answer questions?

Imagine asking: "Show me all my books tagged #reading, sorted by status, with how many pages each has." Without Bases, you'd scroll through folders manually. With Bases, you get a live table that updates the moment you add a new book note.

Obsidian Bases turns your vault into something like a personal spreadsheet — but the rows are your notes, and the data is already in your frontmatter. The obsidian-bases skill teaches AI to build these views for you.

📝
Your notes have properties

Frontmatter like status, tags, date, author — all stored in every note you create with the markdown skill.

🔍
A .base file writes the query

Like a saved search — "give me all notes tagged #book where status != done, sorted by date."

📊
Obsidian renders it live

Open the .base file and see a table, card grid, or list — updated automatically as notes change.

Anatomy of a .base file

A base file is written in YAML — think of it as a form you fill out: "filter by these rules, display these columns, group by this property."

CODE 
filters:
  or:
    - file.hasTag("book")
    - file.hasTag("article")

formulas:
  status_icon: 'if(status == "done", "✅", "📚")'

views:
  - type: table
    name: "Reading List"
    order:
      - file.name
      - author
      - formula.status_icon
PLAIN ENGLISH

Start of filter rules — which notes to include

"Or" means include a note if ANY of these conditions are true

Include notes tagged #book

OR include notes tagged #article

Formulas section — computed columns you can invent

If the note's status is "done", show ✅, otherwise show 📚

Views section — how to display the results

Display as a table (other options: cards, list, map)

The table's name, shown as a tab in Obsidian

Which columns to show, left to right

Column 1: the note's filename

Column 2: the author property from frontmatter

Column 3: the computed status icon formula

Filters — the smart search that remembers itself

Filters are rules that decide which notes appear. Unlike a one-off search you type and forget, a filter in a base file is permanent — saved, reusable, and always up to date.

file.hasTag("book") Include notes with the #book tag
status == "done" Only notes where the status property equals "done"
status != "done" Exclude done notes — show everything in progress
priority > 3 Only high-priority items (where priority is a number)
file.inFolder("Books") Only notes inside the Books folder
file.mtime > today() - "7d" Notes modified in the last 7 days

Filters can be combined with and: (all must be true), or: (any can be true), and not: (exclude matches). Think of them like a smart shopping cart: AND adds conditions, OR broadens the search, NOT removes things you do not want.

Formulas — computed columns you invent

Formulas let you calculate values that do not exist yet in your frontmatter. Think of it like adding a new column to a spreadsheet that computes its value from other columns.

CODE 
formulas:
  days_until_due:
    'if(due, (date(due) - today()).days, "")'

  priority_label:
    'if(priority == 1, "🔴 High",
     if(priority == 2, "🟡 Med", "🟢 Low"))'

  days_old:
    '(now() - file.ctime).days'
PLAIN ENGLISH

Define custom computed columns here

Column named "days_until_due"

If the note has a due property, calculate days remaining; otherwise show blank

Column named "priority_label"

If priority is 1, show 🔴 High; if 2, show 🟡 Med; otherwise 🟢 Low

Column named "days_old"

Subtract the file creation time from now, convert to days

⚠️
Common gotcha: date subtraction returns a Duration, not a number

When you subtract two dates in a formula, you get a Duration object — not a plain number. Always add .days to get the number: (date(due) - today()).days not just (date(due) - today()). This is the #1 formula error the skill prevents.

Visual Canvases

The json-canvas skill — teaching AI to build infinite visual boards where ideas connect in 2D space.

Some ideas do not fit in a line

Notes are linear — one paragraph after another, top to bottom. But some thinking is inherently spatial: project relationships, research maps, system diagrams, concept connections. For those, you need Obsidian Canvas.

NOTES (linear)

CANVAS (spatial)

Idea A
Idea B
Idea C

A canvas is stored as a JSON file with the .canvas extension. The json-canvas skill teaches AI to generate valid JSON that Obsidian can render as a visual board — without you needing to write a single line of JSON yourself.

What a canvas file actually contains

Think of a canvas file like a stage direction script for a play. It says: "Place Actor A at position X, facing Y. Draw a line from Actor A to Actor B." Obsidian reads the script and renders the stage.

CODE 
{
  "nodes": [
    {
      "id": "6f0ad84f44ce9c17",
      "type": "text",
      "x": 0, "y": 0,
      "width": 400, "height": 200,
      "text": "# Hello World"
    }
  ],
  "edges": [
    {
      "id": "0123456789abcdef",
      "fromNode": "6f0ad84f44ce9c17",
      "toNode": "a1b2c3d4e5f67890",
      "label": "leads to"
    }
  ]
}
PLAIN ENGLISH

The entire canvas is one JSON object with two lists

"nodes" is the list of things on the board

Each node is its own object with properties

A unique 16-character ID — no two nodes can share this

"text" means it holds markdown text (other types: file, link, group)

Position on the infinite canvas — x goes right, y goes down

How big the box is in pixels

The markdown content displayed inside the box

"edges" is the list of connecting lines between nodes

Each edge also gets a unique ID

fromNode: the ID of the node where the arrow starts

toNode: the ID of the node where the arrow points

Optional text label that appears on the connecting line

Four types of canvas card

Every item on a canvas is a node. The type property controls what the node displays.

📝

text

A plain text or markdown card. Write headers, lists, callouts — anything markdown supports. Most common type.

📄

file

Displays an existing note from your vault as a preview card. Clicking opens the note. Can show a specific heading with subpath.

🌐

link

Embeds an external URL — shows a web page preview card on the canvas. Great for research boards.

📦

group

A container that visually groups other nodes. Like drawing a box around related cards. Has a label and optional background.

⚠️
Every ID must be unique — and the skill enforces this

The biggest canvas bug is two nodes with the same ID. The skill instructs AI to generate fresh 16-character hex IDs for every node and edge, and to verify that fromNode/toNode references actually exist in the nodes list before writing the file.

05

Power Tools

obsidian-cli and defuddle — AI that can act on your vault and read the web cleanly.

Reading vs. acting

The first four skills taught AI to write content in the right format. The power tools go further: they let AI act directly on a live Obsidian vault, and read from the web without the noise.

⚡

obsidian-cli

A command-line tool that talks directly to a running Obsidian. AI can read, create, search, append to, and manage notes — without you touching a file manually.

🧹

defuddle

A web-page extractor. Give it a URL — it strips navigation, ads, and clutter, returning just the article content as clean markdown. Saves tokens. Works everywhere.

💡
Why this changes what AI can do for you

Without these tools, AI can only write files you then have to save yourself. With obsidian-cli, AI can append to your daily note, search the vault, create notes silently in the background, and check what already exists — all in one conversation.

obsidian-cli — your vault's remote control

Think of the CLI as a remote control for Obsidian. Instead of clicking through menus, AI types commands that Obsidian executes immediately. Obsidian must be open for this to work.

obsidian read file="My Note" Read the full content of a note by name
obsidian create name="New Note" content="# Hello" silent Create a new note without opening it in Obsidian (silent = background)
obsidian append file="My Note" content="- [ ] New task" Add content to the end of an existing note — perfect for daily notes
obsidian search query="project alpha" limit=10 Search the vault and return matching notes (up to 10 results)
obsidian daily:append content="- Meeting at 3pm" Append directly to today's daily note without specifying the filename
obsidian property:set name="status" value="done" file="My Note" Update a frontmatter property without rewriting the whole note
obsidian tags sort=count counts List all tags in the vault, sorted by how often they are used
🔧
Plugin development too

The obsidian-cli skill also covers plugin development — reloading plugins, capturing errors, taking screenshots, and running JavaScript in the Obsidian context. Useful when building or debugging Obsidian plugins with AI assistance.

defuddle — the web, without the noise

Web pages are designed for humans: headers, footers, sidebars, cookie banners, ads, related articles. For AI reading a page, this noise wastes tokens and buries the actual content. Defuddle acts like a surgeon — it removes everything except the article body.

CODE 
# Basic usage — get clean markdown
defuddle parse <url> --md

# Save to a file in your vault
defuddle parse <url> --md -o content.md

# Get just the title
defuddle parse <url> -p title

# Get the description (for frontmatter)
defuddle parse <url> -p description
PLAIN ENGLISH

The --md flag outputs clean markdown (not raw HTML)

Fetch the URL and return just the article content as markdown

Same as above but write to a file — great for saving research to vault

-o sets the output filename

-p extracts a single metadata property instead of the full content

Useful for generating frontmatter: grab title, description, domain from the URL

WEBFETCH (raw)

Returns full HTML: navigation, ads, cookie banners, sidebar, footer, comments — thousands of tokens the AI must process before finding the article.

DEFUDDLE (clean)

Returns only the article body as clean markdown. 10-20x fewer tokens. AI reads faster, costs less, and focuses on what matters.

End-to-end: save a web article to your vault

Here is how defuddle and obsidian-cli work together to turn a URL into a properly formatted vault note in one go.

1
You share a URL

"Save this article to my vault: https://example.com/great-article"

2
AI runs defuddle

3
AI composes the note

Uses obsidian-markdown skill to wrap content in frontmatter (title, source URL, date, tags) and format any callouts or highlights.

4
AI creates the vault note

obsidian create name="Article Title" content="..." silent — the note appears in your vault immediately, no manual saving needed.
